# 小猪出栏玩法拆解

## 视频核心玩法

- 核心循环：点击一只动物，动物会沿当前朝向一直向前移动；如果正前方直到边界没有其它动物，就离开游玩区；如果被挡住，则不能移动。
- 目标：把场内所有动物都清出草地区。
- 压力来源：动物方向交错，玩家需要先找出“通路打开”的动物，逐步释放空间。
- 反馈：成功移动时有跑动起伏、沿路扬尘和连击提示；失败时动物会先沿朝向冲到阻挡物前，再撞击并出现眩晕星星。
- 道具：视频底部有移除、洗牌、翻转三类道具。移除和翻转都是先点道具，再点指定动物生效。

## 小羊尺寸观察

- 逻辑占位：每只小羊不是 1 个格子，而是 2 个连续格子。
- 朝左/朝右时占 `2 列 x 1 行`；朝上/朝下时占 `1 列 x 2 行`。
- 数据点按“头所在的格子”记录，身体占头后方的另一个格子。
- 视觉占位：贴图略大于逻辑占位，约为 `1.2 格宽 x 2.1 格长`，边缘会压到相邻格线，接近视频里那种长一点、挤在一起的形状。

## 关卡扩展方式

棋盘尺寸固定写在 `game.js` 的 `BOARD` 中，每一关共用同一个棋盘：

```js
const BOARD = {
  cols: 12,
  rows: 19,
};
```

关卡集中写在 `LEVELS` 数组里。关卡不再改变棋盘大小，而是通过 `playArea`、动物数量、动物密度和朝向关系控制难度：

```js
{
  id: 1,
  name: "第1关",
  playArea: { x: 1, y: 1, cols: 10, rows: 13 },
  animals: [
    { x: 1, y: 1, dir: "left" },
  ],
}
```

后续增加难度时，保持 `BOARD` 不变，逐步扩大本关实际使用的 `playArea`，并增加动物数量、密度和方向交错程度。

关卡摆放需要规避同一行或同一列里的对向动物，例如同一列上方朝下、下方朝上。这种组合会互相顶住，属于天然死锁。

后台关卡编辑器是独立页面 `level-editor.html`，不挂在玩家主界面里。编辑器用于生成、手动摆放、删除、调整朝向、查看流量风险，并复制可贴回 `LEVELS` 的关卡数据。

同时需要控制“出口流量”，避免出现一整行或一整列动物可以连续直接跑出，或移除一只动物后瞬间打开大量通路。`game.js` 中的 `FLOW_RULES` 会检查：

- 初始可直接出场动物数量。
- 同一出口方向可直接出场动物数量。
- 同一出口线可连续跑出的链条长度。
- 移除单只动物后新增的可出场动物数量和最长出口链。

当前已有四关：

- 第2关：68 只小猪，使用完整棋盘区域，作为较难的密集关卡保留。

## 计分与星级

- 小猪成功跑出时得分：基础 `100` 分，连击每多 1 段额外增加 `50` 分。
- 例如连续跑出第 1 只得 `100`，第 2 只得 `150`，第 3 只得 `200`。
- 撞晕会打断连击，下一只重新从 `100` 分开始。
- 使用移除、翻转道具不会打断连击；道具本身不直接给高分，主要价值是打开通路、延长后续连走链条。
- 清空本关全部小猪就算通关，并至少获得一星；一星不设置分数门槛。
- 二星线按关卡动物数量自动计算，并假设需要大致达到平均 `16` 连走水平。
- 三星线是本关理论最高分，也就是所有小猪都不中断连击、全部跑出时的满分。
- 每关会记录历史最高星级和最高分；总星数等于所有关卡历史最高星级之和，后续可作为购买道具的资源。
- 玩家可以通过 HUD 的关卡选择重新挑战任意关卡，补没拿到的二星或三星。

当前第4关的星级线是：

- 一星：通关
- 二星：`28500`
- 三星：`94500`

## 道具交互

- 移除：先点击底部“移除”，再点击棋盘中的一只动物，这一只动物消失。
- 翻转：先点击底部“翻转”，再点击棋盘中的一只动物，这一只动物在原地前后翻转，原来的头格变成尾格，原来的尾格变成头格。

## 动物特效配置

动物类型集中写在 `game.js` 的 `ANIMAL_TYPES` 中。后续增加不同动物时，可以单独调整：

```js
pig: {
  moveMsPerCell: 115,
  crashMoveMsPerCell: 170,
  crashBumpMs: 160,
  stunMs: 620,
  trailColor: "rgba(241, 218, 178, 0.82)",
  burstColor: "rgba(255, 255, 255, 0.86)",
  stunColor: "#ffe76a",
}
```

关卡中的动物可通过 `type` 指定类型；不写时使用关卡默认的 `animalType`。
